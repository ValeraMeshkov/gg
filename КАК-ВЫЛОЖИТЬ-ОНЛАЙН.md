# Как играть вдвоём через интернет (простыми словами)

У вас **две вещи**, как у сайта с чатом:

| Что | Где | Зачем |
|-----|-----|--------|
| **Сайт с игрой** | GitHub Pages | Открывается в браузере |
| **Сервер** | Render.com (бесплатно) | Создаёт комнаты, синхронизирует ходы |

**GitHub Pages сервером быть не может** — вы правильно поняли.  
**Render** — это просто сайт, где крутится ваш `server/` (как «включённый всегда npm run dev:server» в облаке).

---

## Часть 1. Сервер на Render (15 минут, один раз)

### Шаг 1. Регистрация

1. Откройте **https://render.com**
2. **Get Started** → войдите через **GitHub** (тот же аккаунт, где репозиторий `gg`)

### Шаг 2. Новый сервис

1. Нажмите **New +** → **Web Service**
2. Подключите репозиторий **`ValeraMeshkov/gg`** (если не видно — **Configure account** и дайте доступ к репо)
3. Заполните поля **точно так**:

| Поле | Значение |
|------|----------|
| **Name** | `gg-game` (любое имя) |
| **Region** | Frankfurt или ближайший |
| **Branch** | `main` |
| **Root Directory** | `server` |
| **Runtime** | Node |
| **Build Command** | `npm install && npm run build` |
| **Start Command** | `npm start` |
| **Instance Type** | **Free** |

### Шаг 3. Одна важная настройка

Прокрутите до **Environment Variables** → **Add Environment Variable**:

| Key | Value |
|-----|--------|
| `CORS_ORIGINS` | `https://valerameshkov.github.io` |

(без `/gg` в конце)

### Шаг 4. Запуск

1. Нажмите **Create Web Service**
2. Подождите 3–5 минут, пока статус станет **Live** (зелёный)
3. Вверху скопируйте URL, например:  
   `https://gg-game.onrender.com`

### Шаг 5. Проверка

Откройте в браузере (подставьте свой URL):

`https://gg-game.onrender.com/api/health`

Должно быть: `{"ok":true,...}`  
Если видите — сервер работает.

## Часть 2. Связать сайт на GitHub с сервером (5 минут)

### Способ проще всего — один файл

1. На GitHub откройте:  
   **https://github.com/ValeraMeshkov/gg/edit/main/public/api-config.json**
2. Замените содержимое на (вставьте **свой** URL с Render):

```json
{
  "apiBaseUrl": "https://gg-game.onrender.com"
}
```

3. Внизу **Commit changes** → **Commit directly to main**

Подождите 2 минуты (Actions пересоберёт сайт) и откройте:

**https://valerameshkov.github.io/gg/room**

Нажмите **Создать комнату** — должно сработать без красной ошибки.

---

## Как играть

1. Вы: **https://valerameshkov.github.io/gg/room** → **Создать комнату**
2. Копируете ссылку другу (типа `…/gg/room/ABCD`)
3. Друг открывает у себя на телефоне/компе
4. Вы жмёте **Начать** (когда оба в комнате)

---

## Если что-то не работает

| Проблема | Что сделать |
|----------|-------------|
| Render «Build failed» | Напишите в чат — посмотрим лог |
| «Сервер не настроен» на сайте | Проверьте `api-config.json` — правильный URL, без `/` в конце |
| Долго грузится первый раз | На бесплатном Render сервер «просыпается» ~30 сек после сна |
| CORS / ошибка в консоли | В Render переменная `CORS_ORIGINS` = `https://valerameshkov.github.io` |

## Локально (без Render)

Два терминала на своём Mac — как раньше:

```bash
npm run dev:server
npm run dev
```

Открыть http://localhost:5174 — для проверки у себя дома.
